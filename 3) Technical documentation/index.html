Introduction to HTML and CSS

What is HTML (HyperText Markup Language)

HTML (HyperText Markup Language) is the markup language used to create the basic structure of web pages. It consists of a series of elements that define the content of a page, like headings, paragraphs, images, links, and more. Each element is enclosed in tags, which tell the browser how to display the content.


          <h1>This is a heading</h1>
          <p>This is a paragraph.</p>

HTML elements dont define the style of the content, only its structure. The goal of HTML is to provide clear semantics, useful for search engines and users, especially those using assistive technologies.

What is CSS (Cascading Style Sheets)

CSS (Cascading Style Sheets) is the language used to define the style of an HTML document. CSS allows control over the appearance of elements on the page, such as colors, fonts, layout, and spacing. This separates structure (HTML) from style, making it easier to update or modify a website's look without altering its structure.


          h1 {
            color: blue;
            font-size: 36px;
          }
          p {
            font-family: Arial, sans-serif;
            line-height: 1.5;
          }

CSS also enables the creation of complex layouts and responsive websites, which adapt to various screen sizes.

Difference between HTML and CSS

HTML and CSS work together to create web pages, but serve different purposes:

  • HTML handles the structure and content of the page. It's like the skeleton of a building, providing a strong foundation.
  • CSS manages the style and aesthetic appearance of the page. It's like decoration and furnishing, making the site visually appealing and easy to navigate.

How they work together

To see HTML and CSS in action, consider this example of a web page using both:


          <!DOCTYPE html>
          <html lang="en">
            <head>
              <meta charset="UTF-8">
              <meta name="viewport" content="width=device-width, initial-scale=1.0">
              <title>Sample Page</title>
              <link rel="stylesheet" href="styles.css">
            </head>
            <body>
              <h1>Welcome to my site!</h1>
              <p>This is a simple example of how HTML and CSS work together.</p>
            </body>
          </html>

In the CSS file (styles.css), we might have something like:


          body {
            background-color: #f0f0f0;
          }

          h1 {
            color: darkblue;
            text-align: center;
          }

          p {
            font-size: 18px;
            color: #333;
          }

This way, we can modify the appearance of the page without touching the HTML file, making website maintenance simpler and more efficient.

HTML

2.1 Basic Structure of an HTML Document

A basic HTML document follows a specific structure that tells the browser how to interpret the content. Heres the minimum structure of an HTML file:


          <!DOCTYPE html>
          <html lang="en">
            <head>
              <meta charset="UTF-8">
              <meta name="viewport" content="width=device-width, initial-scale=1.0">
              <title>Page Title</title>
            </head>
            <body>
              <!-- Page content goes here -->
            </body>
          </html>
  • <!DOCTYPE html>: Specifies the document type (HTML5).
  • <html>: The root element that contains the entire page content.
  • <head>: Contains meta-information about the page (e.g., charset, viewport, title).
  • <body>: Contains the visible content of the page.

2.2 Common HTML Tags

Below is a list of the most common HTML tags used to create the content of a webpage:

Headings

Headings hierarchically define the page content:


          <h1>Main Heading</h1>
          <h2>Second Heading</h2>
          <h3>Third Heading</h3>

Paragraphs

For defining blocks of text:


          <p>This is a paragraph.</p>

Images

Images are inserted using the <img> tag and require the src (source) attribute to specify the image file:


          <img src="image.jpg" alt="Image description">

Links

For creating hyperlinks, use the <a> tag with the href attribute that specifies the link destination:


          <a href="https://www.example.com">Visit the site</a>

Lists

Lists can be ordered (<ol>) or unordered (<ul>):


          <ul>
            <li>Item 1</li>
            <li>Item 2</li>
          </ul>

          <ol>
            <li>First Item</li>
            <li>Second Item</li>
          </ol>

Tables

Tables are created using the <table>, <tr> (row), <td> (cell), and <th> (header):


          <table>
            <tr>
              <th>Header 1</th>
              <th>Header 2</th>
            </tr>
            <tr>
              <td>Cell 1</td>
              <td>Cell 2</td>
            </tr>
          </table>

Forms

Forms allow users to submit data through inputs such as text fields and buttons:


          <form action="process.php" method="post">
            <input type="text" name="name" placeholder="Enter your name">
            <button type="submit">Submit</button>
          </form>

Divisions and Sections

The <div> tag is used for grouping content and has no semantic meaning, while tags like <section> or <article> are used to create sections of semantic content: 


          <div>
            This is a div.
          </div>

          <section>
            This is a section of content.
          </section>

2.3 HTML Attributes

Attributes provide additional information about HTML elements. They are placed within the opening tag and add extra characteristics to the element.

  • id: A unique identifier for an HTML element.
  • class: Assigns one or more classes to an element, useful for applying CSS styles or referencing the element in JavaScript.
  • style: Applies inline CSS styles directly to the element.
  • src: Used to specify the source of an image (<img>) or a script (<script>).
  • href: Used with the <a> tag to specify the destination of a link.

2.4 HTML Semantics

Using semantic tags makes HTML code clearer, both for search engines and for users who use assistive technologies. Semantic tags give meaning to content, improving accessibility and SEO (Search Engine Optimization).

  • <header>: Defines the header of a page or section.
  • <footer>: Defines the footer of a page or section.
  • <nav>: Defines a section containing navigation links.
  • <main>: Defines the main content of a document.
  • <article>: Defines an article, which can stand on its own from the rest of the page.
  • <section>: Defines a general section of a document.
  • <aside>: Contains secondary content, like a sidebar or related information.
CSS

3.1 Introduction to CSS

CSS (Cascading Style Sheets) is the language used to define the style of an HTML document. It controls the visual appearance of a website, including colors, fonts, layout, and more. CSS separates the content structure (HTML) from its presentation, making site management and maintenance easier.

An example of a CSS rule that applies styling to a heading and a paragraph:


          h1 {
            color: blue;
            font-size: 36px;
          }
          p {
            font-family: Arial, sans-serif;
            line-height: 1.5;
          }

You can link an external CSS file to the HTML document using the <link> tag in the <head> section of the HTML file:


          <link rel="stylesheet" href="styles.css">

3.2 CSS Selectors

Selectors in CSS allow you to specify which HTML elements should be styled. There are several types of selectors:

Simple Selectors

  • Type selector: Selects all elements of a specific type. Example: 
    
          p {
            color: red;
          }
    This applies red color to all <p> elements.
  • Class selector: Selects all elements with a specific class. It uses a period . before the class name: 
    
          .header {
            font-size: 24px;
            color: green;
          }
    This applies styles to all elements with the class header.
  • ID selector: Selects the element with a specific ID. It uses a hash # before the ID name: 
    
          #container {
            padding: 20px;
            background-color: lightgray;
          }
    This applies styles to the element with the ID container.

Combined Selectors

  • Group selectors: Selects multiple elements and applies the same style to them: 
    
          h1, h2, h3 {
            color: navy;
          }
  • Descendant selector: Selects elements that are descendants of another element: 
    
          div p {
            font-size: 18px;
          }
    This applies the style to all <p> elements inside a <div>.

Pseudo-classes and Pseudo-elements

Pseudo-classes and pseudo-elements allow you to style elements based on a particular state or part of the content:

  • Pseudo-class :hover: Applies styles when the user hovers over an element: 
    
          a:hover {
            color: red;
          }
  • Pseudo-element ::before: Inserts content before an element: 
    
          h1::before {
            content: "🎉 ";
          }

3.3 Box Model

The box model in CSS describes how every HTML element is a rectangular box made up of four areas: content, padding, border, and margin.

  • Content: The area that contains the actual content of the element.
  • Padding: The space between the content and the border.
  • Border: The line that surrounds the element.
  • Margin: The outer space that separates the element from others.

Example of CSS rules that define the box model:


          div {
            width: 200px;
            padding: 10px;
            border: 2px solid black;
            margin: 20px;
          }

3.4 Layout with CSS

CSS allows you to manage the layout of the page using various properties. The most commonly used layouts include Flexbox and Grid.

Display

The display of an element determines how it is rendered. Common values include:

  • block: The element occupies the entire available width.
  • inline: The element only takes up the space needed and does not break the text flow.
  • inline-block: The element behaves like an inline element but allows setting dimensions.
  • none: The element is not displayed.

Flexbox

Flexbox is a layout model that allows flexible and centered positioning of elements. Heres an example of a layout using Flexbox:


          .container {
            display: flex;
            justify-content: center;
            align-items: center;
            height: 100vh;
          }

Grid

The CSS Grid Layout allows you to create complex grid-based layouts. Heres an example of a basic layout with Grid:


          .container {
            display: grid;
            grid-template-columns: 1fr 1fr;
            grid-gap: 20px;
          }

3.5 Colors and Measurement Units

CSS supports various methods to specify colors:

  • RGB: rgb(255, 0, 0)
  • HEX: #ff0000
  • HSL: hsl(0, 100%, 50%)

The most common measurement units include:

  • px: Pixels, a fixed unit.
  • %: A percentage relative to the container's size.
  • em and rem: Units relative to the font size.
  • vh and vw: Units relative to the viewport's height and width.

3.6 Text Styling

CSS allows you to control the appearance of text, including fonts, sizes, spacing, and alignment.


          h1 {
            font-family: 'Arial', sans-serif;
            font-size: 36px;
            color: darkblue;
            text-align: center;
          }
          p {
            line-height: 1.6;
            letter-spacing: 0.5px;
          }

3.7 Responsive Design

Responsive design enables you to create layouts that adapt to different screen sizes using media queries and relative units like %, vh, and vw.

An example of a media query for smaller screens:


          @media (max-width: 600px) {
            body {
              font-size: 14px;
            }
          }

Mobile-first design focuses on optimizing for mobile devices first, then expanding the layout for larger screens.

4.4 Creating a Responsive Navigation Menu

Heres an example of a simple navigation menu that becomes responsive using media queries:

HTML


                      <nav>
                        <ul>
                          <li><a href="#">Home</a></li>
                          <li><a href="#">About Us</a></li>
                          <li><a href="#">Services</a></li>
                          <li><a href="#">Contact</a></li>
                        </ul>
                      </nav>

CSS


                      nav ul {
                        list-style-type: none;
                        margin: 0;
                        padding: 0;
                        background-color: #333;
                      }
            
                      nav ul li {
                        display: inline-block;
                        margin-right: 20px;
                      }
            
                      nav ul li a {
                        color: white;
                        text-decoration: none;
                        padding: 10px;
                        display: block;
                      }
            
                      @media (max-width: 600px) {
                        nav ul li {
                          display: block;
                          margin-right: 0;
                        }
                      }

With these rules, the menu automatically adjusts to smaller screens.

Best Practices

5.1 Separating Structure and Style

It is good practice to keep HTML and CSS separate. HTML should define the structure and content, while CSS should handle styling. This approach makes the code easier to maintain and modify over time.


                      <!-- HTML: structure -->
                      <p class="description">This is a paragraph.</p>
            
                      <!-- CSS: style -->
                      .description {
                        font-size: 18px;
                        color: #333;
                      }

5.2 Using Meaningful Class and ID Names

Use descriptive class and ID names to make the code more readable and easier to understand. Avoid generic names like div1 or box2 and prefer names that describe the elements function.


                      <div class="article-content">...</div>

5.3 Avoid Inline Styles

Avoid using inline styles (directly in HTML elements) as it makes the code hard to manage. Instead, use external CSS files or style rules in the <head> section.


                      <!-- Avoid this -->
                      <p style="color: red;">Red text</p>
            
                      <!-- Use this -->
                      <p class="red-text">Red text</p>

5.4 Commenting Your Code

Adding comments in both HTML and CSS helps keep the code organized and easier to understand for you or other developers. Heres an example:


                      <!-- This is an HTML comment -->
                      <p>Page content</p>
            
                      /* This is a CSS comment */
                      p {
                        color: #333;
                      }

5.5 Using the Box Model Correctly

Always set box-sizing to border-box to avoid confusion regarding the size of elements. This ensures that padding and borders are included in the element's size.


                      * {
                        box-sizing: border-box;
                      }

5.6 Optimizing for Performance

Optimize your sites performance by reducing image sizes, compressing CSS and JavaScript files, and minimizing the number of HTTP requests.

Using Optimized Images

Ensure images are saved in the correct format and at an appropriate resolution. For example, use formats like WebP to reduce file sizes without compromising quality.

Minimizing CSS and JS Files

Use tools like cssnano or UglifyJS to reduce the size of your CSS and JavaScript files, improving the loading time of your site.
